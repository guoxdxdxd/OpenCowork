# 提供商配置概述 / Providers Overview

OpenCowork 支持 18+ AI 提供商，通过统一的 `APIProvider` 接口抽象差异。

## 支持的提供商 / Supported Providers

| 提供商                                        | 协议                               | 特性                       |
| --------------------------------------------- | ---------------------------------- | -------------------------- |
| [Anthropic](/docs/providers/anthropic)        | `anthropic`                        | 扩展思考、视觉             |
| [OpenAI](/docs/providers/openai)              | `openai-chat` / `openai-responses` | 视觉、推理                 |
| [DeepSeek](/docs/providers/deepseek)          | `openai-chat`                      | 推理模式                   |
| [Google Gemini](/docs/providers/google)       | `openai-chat`                      | 视觉、长上下文             |
| [Ollama](/docs/providers/ollama)              | `openai-chat`                      | 本地运行，无需 API Key     |
| [其他提供商](/docs/providers/other-providers) | `openai-chat`                      | Qwen、Moonshot、MiniMax 等 |

补充说明：

- Gitee AI 仅支持 API Key 登录方式。
- Codex 支持 OAuth 登录（授权后直接调用 OpenAI 兼容接口）。

## 添加提供商 / Add Provider

1. 进入 **设置 → AI 提供商**
2. 点击 **添加提供商**
3. 选择提供商类型
4. 填入 API Key 和可选的 Base URL
5. 选择默认模型

## 自定义 Base URL

所有 OpenAI 兼容的提供商都支持自定义 Base URL，可以接入任何兼容 OpenAI API 的服务：

```
Base URL: https://your-proxy.example.com/v1
```

## 图像模型 / Image Models

部分提供商支持图像生成模型（如 OpenAI 的 DALL·E 系列）。配置图像模型时，需要将模型的 `category` 设为 `image`：

1. 进入 **设置 → AI 提供商**
2. 在模型列表中找到或添加图像生成模型
3. 确保模型分类为 `image`
4. 启用模型

启用后，可以在 **设置 → 插件** 中开启[图像生成插件](/docs/features/app-plugins)，让 Agent 获得图片生成能力。

## 模型定价 / Pricing

每个模型配置包含定价信息，用于在 UI 中显示预估费用：

```typescript
{
  inputPrice: 3.0,   // 每百万 input tokens 的美元价格
  outputPrice: 15.0, // 每百万 output tokens 的美元价格
}
```
